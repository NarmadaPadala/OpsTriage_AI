# Notebook Adaptation Guide

The GenAcademy Week 5 notebook should be reused as the starting workflow for Qwen3 LoRA fine-tuning, but the tutorial-specific data and evaluation sections must be adapted for OpsTriage AI.

## What Stays the Same

Keep the notebook cells that handle:

- Runtime and package setup
- Hugging Face authentication, if required
- Base model selection
- LoRA training flow
- Adapter checkpoint loading
- Adapter merge or export flow

These cells are infrastructure workflow cells and should not be rewritten unless the installed library versions require updates.

## What Changes

### Dataset Loading

Replace tutorial dataset loading with the processed OpsTriage files:

- `data/processed/splits/train.json`
- `data/processed/splits/validation.json`
- `data/processed/splits/test.json`

These files are generated by:

```bash
python3 scripts/prepare_dataset.py
```

### ShareGPT/OpenAI Format

The OpsTriage dataset uses `messages` records:

```json
{
  "messages": [
    {
      "role": "system",
      "content": "You are OpsTriage AI..."
    },
    {
      "role": "user",
      "content": "Incident Title: ...\nIncident Description: ..."
    },
    {
      "role": "assistant",
      "content": "API Platform"
    }
  ]
}
```

The assistant response must be exactly one approved support-team label.

### LoRA Config

Use the Sprint 3.2 config as the source of truth:

```text
configs/training/llamafactory_qwen3_1_7b_lora_sft.yaml
```

Key parameters:

- Model: `Qwen/Qwen3-1.7B`
- Method: LoRA
- Stage: supervised fine-tuning
- Sequence length: `1024`
- Learning rate: `1e-4`
- Epochs: `4`
- LoRA rank: `16`
- LoRA alpha: `32`
- LoRA dropout: `0.05`

### Evaluation

Replace tutorial metrics with enterprise incident-classification metrics:

- Accuracy
- Precision
- Recall
- Macro F1
- Weighted F1
- Confusion matrix
- Invalid-label rate

Macro F1 should be treated as the primary model-selection metric because all support teams matter, including smaller classes.

### Base vs Fine-Tuned Comparison

The notebook should benchmark:

1. Base Qwen3-1.7B prompted with the same classification prompt.
2. Fine-tuned Qwen3-1.7B LoRA adapter.
3. Merged model, if adapter merge is performed.

The fine-tuned model should not be considered successful unless it improves macro F1 and reduces high-value confusion pairs without increasing invalid labels.

## What to Remove

Remove or skip cells that are specific to:

- Toy examples
- Generic chat completions unrelated to incident routing
- Sentiment, summarization, or open-ended Q&A datasets
- Chain-of-thought generation
- Tutorial-only visualizations that do not support classification evaluation

## What to Add

Add cells for:

- Dataset quality report review
- Label taxonomy check
- Base model benchmark
- Per-class metrics
- Confusion matrix
- Error analysis table
- Model card notes
